# Code Guide

Conventions, patterns, and rules for `kafe-web`.

---

## Commands

```bash
pnpm dev           # Start dev server (localhost:3000)
pnpm build         # Production build — also catches TypeScript errors
pnpm lint          # ESLint with Next.js rules
pnpm generate:api  # Regenerate React Query hooks from OpenAPI spec
                   # (backend must be running at localhost:3333)
```

---

## File naming

| Type | Convention | Example |
|------|-----------|---------|
| Page | `page.tsx` | `app/admin/users/page.tsx` |
| Layout | `layout.tsx` | `app/admin/layout.tsx` |
| Route handler | `route.ts` | `app/api/auth/login/route.ts` |
| Component | `PascalCase.tsx` | `components/admin/UsersTable.tsx` |
| Hook | `camelCase.ts` | `hooks/usePolling.ts` |
| Utility / lib | `kebab-case.ts` | `lib/api/server-fetch.ts` |

Path alias `@/` resolves to the project root.

## Server vs Client components

Default to **Server Components**. Add `'use client'` only when you need:
- React state or effects (`useState`, `useEffect`)
- Browser APIs (`window`, `document`)
- React Query hooks
- Event handlers

```typescript
// ✅ Server Component — no directive needed
export default async function UsersPage() {
  const users = await serverFetch('/api/v1/users', { method: 'GET' })
  return <UsersTable users={users} />
}

// ✅ Client Component — directive required
'use client'
export function UsersTable({ users }: { users: User[] }) {
  const [selected, setSelected] = useState<string | null>(null)
  ...
}
```

## Data fetching

### In Server Components — use `serverFetch`

```typescript
import { serverFetch } from '@/lib/api/server-fetch'

const res = await serverFetch<{ data: User[]; status: number; headers: Headers }>(
  '/api/v1/users',
  { method: 'GET' },
)
```

`serverFetch` reads the `kafe_token` cookie from the request and adds the `Authorization` header automatically.

### In Client Components — use generated hooks

```typescript
import { useUsersControllerList } from '@/lib/api/generated/api'

const { data, isLoading } = useUsersControllerList()
```

Hooks are generated by Orval from the backend's OpenAPI spec. Run `pnpm generate:api` to update them.

### Manual server-side modules (`lib/api/*.ts`)

Use these for pages that need server-side fetching but whose data shape isn't covered by the generated hooks, or when you need to compose multiple calls:

```
lib/api/categories.ts   getCategories(), getCategoryById()
lib/api/dashboard.ts    getSummary(), getTopProducts(), getPeakHours()
lib/api/inventory.ts    getIngredients(), getIngredient(), getMovements()
lib/api/orders.ts       getMyOrders()
lib/api/products.ts     getProducts(), getProduct()
lib/api/users.ts        getUsers(), getUser()
```

## Styling

Always merge Tailwind classes with `cn()`:

```typescript
import { cn } from '@/lib/utils'

<div className={cn('base-classes', condition && 'conditional-class', className)} />
```

Never concatenate class strings manually — `cn()` handles deduplication and merge conflicts via `tailwind-merge`.

## UI components

Check `components/ui/` before building any new primitive. Available:

`badge` · `button` · `dialog` · `input` · `pagination` · `select` · `skeleton` · `sonner`

Do not add new primitives to `components/ui/` unless they are genuinely absent — they come from Shadcn and are imported or customised there.

Icons: always use `lucide-react`.

---

## Forms

Use `react-hook-form` + `zod`:

```typescript
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

const schema = z.object({ name: z.string().min(2) })

const form = useForm({ resolver: zodResolver(schema) })
```

## Toasts

```typescript
import { useToast } from '@/context/ToastContext'

const { addToast } = useToast()
addToast('Saved successfully', 'success')
// types: 'success' | 'error' | 'warning' | 'info'
```

## No `any`

Never use `any`. Use explicit types, `unknown` with narrowing, or the generated types from `lib/types/index.ts`.

```typescript
// ❌
const res: any = await fetch(...)

// ✅
const res = await serverFetch<{ data: User[]; status: number; headers: Headers }>(...)

// ✅ for unknown error shapes
catch (err: unknown) {
  const message = err instanceof Error ? err.message : 'Unknown error'
}
```

## Contexts

| Context | Hook | Provides |
|---------|------|---------|
| `AuthContext` | `useAuth()` | `{ user, setUser, logout }` |
| `CartContext` | `useCart()` | `{ items, total, itemCount, addItem, removeItem, updateQuantity, clearCart }` |
| `ToastContext` | `useToast()` | `{ addToast(message, type) }` |

Cart state is persisted in `sessionStorage` (key: `kafe_cart`).
